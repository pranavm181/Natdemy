## Root (`/`)

| Item | Purpose |
| --- | --- |
| `pubspec.yaml` | Declares Flutter/Dart dependencies, assets, fonts, app metadata. |
| `pubspec.lock` | Locked versions of all dependencies in use. |
| `analysis_options.yaml` | Dart analyzer & lint rules. |
| `README.md` | Project overview / setup notes. |
| `API_DATA_REQUIREMENTS.md` | Detailed backend data contract (all endpoints & fields). |
| `API_FIELDS_SUMMARY.md` | Snapshot of which API fields the app already uses. |
| `API_PRIORITY_REQUIREMENTS.md` | Ranked list of backend features required by the app. |
| `MISSING_API_FEATURES.md` | Detailed backlog of API gaps vs. app needs. |
| `FOLDER_STRUCTURE_GUIDE.md` | (This document) Map of the file structure and responsibilities. |
| `assets/` | App assets such as logos or fallback course images. |
| `test/` | Holds Flutter tests (currently only the default sample test). |
| `android/`, `ios/`, `macos/`, `windows/`, `linux/` | Platform shells generated by Flutter; modify only for native integrations. |
| `web/` | Web entry point (`index.html`), manifest, icons for PWA builds. |

> **Note:** Build artifacts (`build/`, `.dart_tool/`, etc.) are ignored via `.gitignore`.

## `lib/` ‚Äî Application Source Code

### Entry point
| File | Purpose |
| --- | --- |
| `lib/main.dart` | App entry point. Sets up theming and routes, launches `SplashScreen`. |

### `lib/api/` ‚Äî HTTP layer
| File | Purpose |
| --- | --- |
| `api_client.dart` | Shared HTTP methods (GET/POST/PUT/DELETE), base URL, token headers. |
| `course_service.dart` | Fetches courses from `/api/home/`, filters placeholders, handles errors. |
| `student_service.dart` | Fetches student list & enrolled courses from `/api/students/`. |
| `material_service.dart` | Fetches course materials from API (expects `data.materials`). |
| `contact_service.dart` | Retrieves contact info with caching and default fallback. |

### `lib/data/` ‚Äî Models, persistence, static fixtures
| File | Purpose |
| --- | --- |
| `student.dart` | `Student` model and `ClassItem` (used by classes screen). |
| `course_catalog.dart` | `Course` model plus fallback course list. |
| `joined_courses.dart` | Stores ‚Äújoined‚Äù courses in SharedPreferences; can sync via API. |
| `auth_helper.dart` | Saves/loads auth state (name/email/token) locally. |

| `materials_config.dart` | Legacy hardcoded PDFs used if API fails. |
| `lessons_config.dart` | Hardcoded map of chapters ‚Üí lessons (placeholder until API is wired). |
| `lesson_videos_config.dart` | Hardcoded video metadata (placeholder until API is wired). |

### `lib/utils/` ‚Äî Helpers

## How to extend

1. **Add a new API call** ‚Üí create a service in `lib/api/`, model in `lib/data/`, update relevant screen.
2. **Add a new screen** ‚Üí create in `lib/screens/`, reuse widgets from `lib/widgets/`.
3. **Persist data** ‚Üí use SharedPreferences patterns in `auth_helper.dart` / `joined_courses.dart`.
4. **Add assets** ‚Üí place under `assets/` and register in `pubspec.yaml`.

This overview should help you quickly locate files and understand how the app is organized. Let me know if you need deeper detail on any specific area!
